How to read the map

The key distinction is between frontier value and deployability. A method can be statistically impressive and still be a poor default choice if software support, diagnostics, or real-world use remain thin.

Deployability = software + empirical support + scope Frontier = novelty + bias handling + decision lift Priority = 55% frontier + 45% deployability

Quick read

  • Closest to routine use: narrow small-sample fixes, exact-likelihood sparse-data tools, and RoBMA-style Bayesian bias adjustment.
  • Most strategically important frontier: target-population transportability and survival IPD network synthesis.
  • Still sensitivity-analysis territory: non-normal random-effects models and multi-indication borrowing.
  • Most experimental: individualized treatment-rule synthesis and RCT plus observational integration frameworks.
