Project Plan

Meta-Frontier Readiness Atlas

This project turns the recent methods scan into a decision-facing HTML atlas: which frontier meta-analysis methods are usable now, which are promising but still fragile, and which remain genuinely research-stage.

Standalone HTML app Weighted readiness scoring Primary-source links WebR cross-check

Scope

  • Cover the most relevant 2024 to 2026 frontier methods from statistics and methods journals.
  • Rank methods by deployability and frontier value rather than novelty alone.
  • Separate narrow-use but ready tools from broad but still experimental frameworks.
  • Keep the scoring rubric visible so the ranking is auditable inside the page.

Core Output

  • Top-line summary cards
  • Ranked method deck
  • Readiness bands
  • Source table with direct paper links

Scoring Logic

  • Deployability = 0.45 software + 0.35 empirical + 0.20 scope
  • Frontier score = 0.45 novelty + 0.35 bias handling + 0.20 decision lift
  • Priority score = 0.55 frontier + 0.45 deployability

Method Set

Small-k robust variance methods Exact sparse-data methods P-hacking-aware synthesis RoBMA-regression Dependent-effect bias correction Transportability with IPD RCT plus observational integration IPD survival NMA Individualized treatment-rule synthesis Non-normal random effects Multi-indication HTA synthesis

WebR Validation Plan

  • Recompute weighted scores in base R from the embedded method data.
  • Compare top ranked method IDs between JavaScript and R.
  • Compare the mean deployability score.
  • Compare the counts inside each readiness band.

The WebR panel is reviewer-facing only and does not drive the page logic.

Build Order

  • 1. Freeze the method set and the scoring inputs.
  • 2. Implement ranking and banding in plain JavaScript.
  • 3. Build the visual atlas with summary cards and source references.
  • 4. Add the WebR panel and R script preview.
  • 5. Place the project in C:\Projects with index.html and this plan.
